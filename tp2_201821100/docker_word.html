도커 용어

1. 도커파일(Docker File)

  • 도커파일은 이미지 파일과 설정값으로 빌드 하여서 설정한 이미지 파일을 생성합니다.
  • 도커 파일은 빌드가 네트워크 불안정하거나 시스템 라이브러리 보안 패치 등으로 이미지 생성에 영향을 줄수 있어서 빌드가 항상 성공하는 것을 보장하지 못합니다.
  • 그래서 도커파일을 작성 시 주로 테스트 주도 개발방식으로 만듭니다.

2. 이미지(Docker Image)

  • 이미지 파일은 컨테이너 실행에 필요한 파일과 설정값을 포함하고 있는 것으로 상태 값을 가지지 않고 변하지 않습니다.
  • 컨테이너는 이미지를 실행한 상태이며, 추가되거나 변하는 값은 컨테이너에 저장됩니다.
  • 같은 이미지로 여러 개의 컨테이너를 생성할 수 있고 컨테이너 상태가 바뀌거나 컨테이너가 삭제해도 이미지는 변하지 않습니다.

3. 컨테이너(Docker Container)

  • 컨테이너는 코드와 모든 종속성을 패키지화하는 소프트웨어의 표준 단위입니다.
  • 개별 소프트웨어의 실행이 필요한 실행환경을 독립적으로 운용할 수 있도록 기반환경 또는 다른 실행환경과의 간섭을 막고 실행의 독립성을 확보해주는 운영체계 수준의 격리 기술을 말합니다.
  • 애플리케이션을 실제 구동 환경으로 부터 추상화 할 수 있는 논리 패키징 메커니즘을 제공합니다.
  • 가상화 기술이지만, 운영체제를 사용하는 기존의 가상화 방식보다 경량화된 방식입니다.
  • 운영체제를 설치하지 않으며, 하드웨어를 가상화하는 계층없이 호스트의 자원을 직접 이용해 메모리 접근, 파일 시스템, 네트워크 속도가 가상머신에 비해 빠릅니다.
